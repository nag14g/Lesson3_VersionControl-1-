Highlight #1 Version Control

Version control is a type of software that records and keeps track of different versions of a single file or directory. Developers use version control in order to collaborate or edit a single file at the same time, send code to another developer who needs it for their work, and to eliminate the chance of someone deleting or losing a certain version due to saving documents with different names. Version control software automatically keeps notes on who changes a file, when was it last edited, shows the most up to date or recently edited version, shows additions and deletions, and merges that occurred.

Highlight #2 Repository

A repository is a file or a directory created in order for a developer to record their changes and trach tier changes in a single area and also see different versions of a single file or directory. A remote repository is a repository that is located on a server and is shared with teams and team members. A local repository is a repository that is located on a single machine of a user. A Benefits of a local repository is you can create a new or clone an existing repository. An example of a local repository is the Git software.

Highlight #3 Git

Git is a type of version control software that allows the user to use a distributed version control meaning that all users get a copy of the complete repository, it is local, there isn’t a single point of failure, it processes faster and it will prevent overwrite of files saved on the repository. Using Git or any version control software helps to avoid having to save different versions of one file under different names like version1.doc version2.doc etc. that can get out of control with the more files you have. Git also links to sources like cloud 9 and can be used to save your information in two spots.

Highlight #4 GitHub

GitHub is a free open source version control software that allows the use of local and remote repositories. It is the largest host for Git repositories. It is widely popular for its ability to host large projects and the use of code review, and issue tracking amongst other things. GitHub is used both by professionals as well as students just starting to learn about Git. Some benefits of GitHub are its simplicity, the use of professional and open source repositories, and allows you to share your code amongst coworkers, classmates or anyone you give access to see what you are working on.

Highlight #5 Commands

When using Git there are some basic and easy to use commands when working with a software like GitHub. Some of the basic and most used Git commands are, Git init, initializes the repository. Git add, adds all the files that were changed since the last back up to the staging area. Git commit – m “…” , commits the changes to the repository. Git push, pushes the changes in the local repository to the remote repository. Git pull, pulls the changes in a remote repository to local repository.
